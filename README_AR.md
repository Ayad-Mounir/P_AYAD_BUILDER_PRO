## 🎯 المشكلة والغرض

AYAD BUILDER PRO هو أداة احترافية لبناء تطبيقات Python مصممة للمطورين الذين يحتاجون إلى إنشاء وتحليل وتجميع مشاريع Python إلى ملفات تنفيذية مستقلة دون الحاجة إلى التعامل مع إدارة المكتبات المعقدة وتكوينات البناء المعقدة. إذا كنت تعاني من مشاكل الاستيراد المخفي في PyInstaller، أو مشاكل الترميز، أو تصحيح الأخطاء في التطبيقات المترجمة، فإن هذه الأداة تزيل هذه المشاكل من خلال تحليل ذكي للمشاريع واكتشاف المكتبات التلقائي وواجهة بناء مرئية. يوفر التطبيق دعماً كاملاً للعربية والإنجليزية، مما يجعله في متناول المطورين في جميع أنحاء العالم.

## ✨ الميزات

- **تحليل المشروع الذكي** — يقوم بفحص تلقائي لجميع أجزاء كود Python الخاص بك لتحديد جميع المكتبات والاستيرادات المخفية وتكوينات المكتبات دون الحاجة إلى تكوين يدوي
- **دعم متعدد اللغات** — واجهة عربية/إنجليزية كاملة مع تبديل لغات فوري عبر فئة `Translator` لتوطين سلس
- **تشخيص الملفات التنفيذية** — نظام تشخيص exe مدمج يحدد مشاكل وقت التشغيل ويقترح إصلاحات من خلال دالة `diagnose_exe()`
- **تحليل ملفات PE** — تحليل شامل وفحص بنية الملف التنفيذي بما في ذلك الاستيرادات والصادرات والموارد ومعلومات التصحيح باستخدام معالجة تنسيق PE الصناعية
- **نظام بناء محمي** — آليات حماية وإخفاء دمجة موثقة في PROTECTION_GUIDE.md لتأمين تطبيقاتك المترجمة
- **معالجة وحدة التحكم UTF-8** — إصلاح تلقائي لترميز وحدة تحكم Windows لإزالة أعطال Unicode/الترميز في التطبيقات المترجمة
- **إدارة بيئة افتراضية** — إنشء وإدارة venv مدمجة من خلال `VenvManager` للتعامل المعزول للمكتبات
- **استرجاع الأخطاء والتسجيل** — معالجات استثناء عالمية مع تسجيل احتياطي آمن عند إغلاق stdout/stderr في الوضع المرئي

## 🚀 التثبيت

### الطريقة 1: إعداد سريع (موصى به)

1. **قم بتحميل المشروع** من GitHub أو قم باستخراج مجلد AYAD_BUILDER_PRO
2. **افتح Command Prompt** (Windows) أو Terminal (macOS/Linux) في مجلد المشروع
3. **قم بتشغيل الإعداد**:
   ```bash
   pip install -r requirements.txt
   ```
4. **تحقق من التثبيت**:
   ```bash
   python main.py --version
   ```

### الطريقة 2: إعداد يدوي مع بيئة افتراضية

1. **أنشئ بيئة افتراضية**:
   ```bash
   python -m venv .ayad_build_venv
   ```
2. **فعّل البيئة**:
   - **Windows**: `.ayad_build_venv\Scripts\activate`
   - **macOS/Linux**: `source .ayad_build_venv/bin/activate`
3. **ثبّت المكتبات**:
   ```bash
   pip install -r requirements.txt
   ```

## 📖 دليل الاستخدام

### بدء التطبيق

**مستخدمو Windows**: انقر نقراً مزدوجاً على `main.py` لتشغيل الواجهة الرسومية.

**جميع الأنظمة الأخرى**: افتح محطة طرفية في مجلد المشروع وقم بتشغيل:
```bash
python main.py
```

سيقوم التطبيق بـ:
- تهيئة وحدة التحكم مع ترميز UTF-8 (تحسين Windows)
- التحقق من تثبيت PyQt6 وعرض خيار التثبيت إذا كان مفقوداً
- عرض نافذة التطبيق الرئيسية

### 🔍 سير العمل الأساسي: بناء مشروع Python إلى ملف EXE

#### الخطوة 1: تحميل مشروعك
1. انقر على **File** → **Open Project** في شريط القوائم
2. انتقل إلى مجلد مشروعك (المجلد الذي يحتوي على سكريبت main الخاص بك)
3. انقر على **Select Folder**
4. سيقوم التطبيق بفحص جميع ملفات `.py` وعرض معلومات المشروع

#### الخطوة 2: راجع تحليل المشروع
بعد التحميل، سترى:
- **المكتبات المكتشفة** — جميع المكتبات المستوردة (مثل `requests` و `numpy` و `PyQt6`)
- **الاستيرادات المخفية** — الوحدات المكتشفة التي عادة ما يتجاهلها PyInstaller
- **نقطة الدخول** — سكريبت Python الرئيسي المحدد افتراضياً
- **شجرة المشروع** — جميع الملفات المصدرية منظمة هرمياً

⚠️ **تحقق من المكتبات المفقودة** — إذا تم وضع علامة على المكتبات الحرجة باللون الأحمر، ثبّتها أولاً: `pip install library_name`

💡 **نصيحة**: يكتشف المحلل تلقائياً الأنماط الشائعة (Django و Flask والتعلم الآلي) ويقترح التكوينات المناسبة.

#### الخطوة 3: تكوين إعدادات البناء
1. في لوحة **Build Settings**، قم بتعيين:
   - **Output Name** — اسم ملف `.exe` الخاص بك (افتراضي: اسم مشروعك)
   - **Icon** — انقر على أيقونة المجلد لاختيار ملف `.ico` للملف التنفيذي الخاص بك
   - **Console Mode** — برنامج بدون واجهة (غير محدد) أو تطبيق وحدة تحكم (محدد)
   - **One-File** — محدد = ملف `.exe` واحد؛ غير محدد = مجلد مع المكتبات التبعية
2. وسّع **Advanced Options** للـ:
   - **Optimization Level** — المقارنة بين السرعة وحجم الملف
   - **Strip Binaries** — إزالة رموز التصحيح لتقليل الحجم
   - **Custom Hooks** — للمكتبات المعقدة مثل TensorFlow أو OpenCV

⚠️ **متطلب الأيقونة**: استخدم ملفات `.ico` بحجم 256×256؛ يجب تحويل ملفات PNG أولاً.

#### الخطوة 4: تشغيل التشخيص (اختياري لكن موصى به)
1. انقر على **Tools** → **Run Diagnostics**
2. سيقوم النظام بالتحقق من:
   - توافق إصدار Python
   - سلامة تثبيت PyQt6
   - مساحة القرص المتاحة
   - مكتبات C++ المفقودة
3. أصلح أي تحذيرات ⚠️ قبل المتابعة

#### الخطوة 5: ابدأ البناء
1. انقر على زر **Build Project** (الزر الأخضر مع أيقونة المطرقة)
2. راقب شريط التقدم بينما يقوم النظام بـ:
   - جمع جميع المكتبات التبعية
   - حل الاستيرادات المخفية
   - تجميع bytecode Python
   - ربط الملف التنفيذي
3. نافذة سجل البناء تعرض الإخراج التفصيلي (راقب تحذيرات ⚠️)

#### الخطوة 6: اختبر ملفك التنفيذي
1. بعد الانتهاء، انقر على **Open Output Folder** لعرض ملف `.exe` المترجم
2. انتقل إلى مجلد الإخراج (عادة ما يكون `dist/` أو `build/`)
3. انقر نقراً مزدوجاً على ملف `.exe` لاختبار تطبيقك
4. إذا حدثت أخطاء:
   - انقر على **Tools** → **Analyze Error** في التطبيق
   - الصق رسالة الخطأ
   - اعرض الإصلاحات المقترحة

### 🔐 تبديل اللغة
1. في قائمة **Preferences**، حدد:
   - **English** — واجهة إنجليزية كاملة
   - **العربية** — توطين عربي كامل
2. تعاد تحميل الواجهة فوراً
3. جميع الحوارات والأزرار ونصوص المساعدة تُحدّث في الوقت الفعلي

### 🛡️ الحماية والإخفاء
للتطبيقات الحساسة أماناً:
1. في **Advanced Options**، حدد **Enable Protection**
2. قم بتكوين مستوى الحماية:
   - **Light** — إخفاء أساسي
   - **Medium** — إخفاء الكود + مكافحة التصحيح
   - **Heavy** — حماية كاملة مع كشف التعديل
3. راجع `PROTECTION_GUIDE.md` للمعلومات الأمنية التفصيلية

### ⚠️ المشاكل الشائعة والإصلاحات

| المشكلة | الحل |
|-------|------|
| **لم يتم العثور على PyQt6** | انقر على زر التثبيت التلقائي أو قم بتشغيل: `pip install PyQt6` |
| **"AttributeError: module has no attribute"** | قم بتشغيل **Tools** → **Analyze Error** واتبع الاقتراحات |
| **يتعطل الملف التنفيذي عند بدء التشغيل** | تحقق من تبديل **Console Mode**؛ التطبيقات بدون واجهة تحتاج معالجة حدث صحيحة |
| **يفشل البناء مع "hidden import not found"** | أضف إلى Advanced Options → Custom Hooks: `--hidden-import=module_name` |
| **حجم الملف كبير (>500MB)** | فعّل **Strip Binaries** وعيّن **One-File** إلى false |
| **Windows Defender يحتفظ بـ exe** | هذا أمر طبيعي؛ وقّع الملف التنفيذي أو أضف استثناء في Defender |

### 💡 أفضل الممارسات

- **اختبر محلياً أولاً**: قم دائماً بتشغيل سكريبت Python الخاص بك مع `python script.py` قبل البناء
- **جمّد المكتبات التبعية**: أنشئ ملف متطلبات مثبتة: `pip freeze > pinned.txt`
- **استخدم المسارات النسبية**: تجنب المسارات المطلقة المشفرة مباشرة؛ استخدم `os.path.dirname(__file__)`
- **قلّل المكتبات التبعية**: أزل الاستيرادات غير المستخدمة لتقليل حجم الملف التنفيذي
- **رقّم إصداراتك**: أدرج معلومات الإصدار في اسم الإخراج (مثل `MyApp_v1.2.1.exe`)

- سلامة نظام البناء
- تبديل اللغة
